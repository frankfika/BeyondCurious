## 二、AI能做什么：核心能力全景图
### 2.1 数据处理层
- **智能识别**：截图OCR、表格提取、PDF解析
- **数据分析**：财务指标计算、趋势可视化、异常发现
- **自动查询**：API对接、实时数据抓取

### 2.2 内容生成层
- **研报写作**：行业分析、竞品对标、财务模型解读
- **多格式输出**：Word、Excel、PPT、Markdown
- **长文本处理**：Kimi 200万字上下文、Claude 100K+ Token

### 2.3 工作流层
- **批量处理**：文件夹自动分类、批量重命名、格式转换
- **多Agent协作**：章节拆分、并行撰写、质量审核
- **版本管理**：Obsidian笔记联动、变更追踪

> 💡 **核心心法**：把AI当作"永不疲倦的实习生"——它擅长执行，但需要你的判断和指令。

### 模块1：公域资料处理——NotebookLM实战（45分钟）

**适用场景**：行业研究、公开财报分析、竞品监控

**核心操作**：
1. **知识库搭建**
   - 上传招股书、年报、行业报告
   - 自动提取关键信息、生成摘要
   - 多文档交叉验证

2. **智能问答**
   - 基于来源的精准回答（带引用）
   - 多文档对比分析
   - 生成播客/音频摘要

3. **研报初稿生成**
   - 从问答到结构化输出
   - 导出Markdown，进入下一环节

**注意事项**：
- ⚠️ 仅处理公开信息，敏感项目资料不要上传
- 建立"公域-私域"分离的工作习惯

---

### 模块2：私域资料处理——本地化RAG系统（90分钟）

**为什么需要私有化？**
- ND协议限制、项目保密要求
- 避免敏感数据泄露风险
- 构建专属投研知识库

#### 2.1 文件管理与分类（Claude Code）

```
工作流：
本地项目文件夹 → 自动分类整理 → 标准化命名 → 元数据标注
```

**实操演示**：
- 批量整理尽调资料
- 自动提取关键信息生成索引
- 文件权限分级管理

#### 2.2 知识库构建（RAGFlow）

**架构示意**：
```
┌─────────────────────────────────────┐
│           RAGFlow 知识库            │
├─────────────┬─────────────┬─────────┤
│  项目文档区  │  行业资料区  │ 模板区  │
├─────────────┼─────────────┼─────────┤
│ • BP/招股书 │ • 行业报告   │ • 研报  │
│ • 尽调资料  │ • 政策文件   │  模板   │
│ • 会议纪要  │ • 竞品分析   │ • 估值  │
│             │              │  模型   │
└─────────────┴─────────────┴─────────┘
              ↓
         向量化存储 + 语义检索
```

**核心功能**：
- 文档向量化：支持PDF、Word、Excel、图片
- 智能检索：语义理解，不用记文件名
- 引用溯源：回答带原文出处，可验证

**权限设计**：
- 文档级别权限控制
- 不同项目/团队隔离访问

### 模块3：研报撰写工作流——多Agent协作（90分钟）

**传统痛点**：单会话字数过多 → 上下文丢失 → 输出质量下降

**解决方案**：分章节 + 多Agent并行

#### 3.1 撰写逻辑设计

```
研报结构示例：
├── 执行摘要（Agent A）
├── 行业分析（Agent B）
│   ├── 市场规模
│   ├── 竞争格局
│   └── 发展趋势
├── 公司分析（Agent C）
│   ├── 业务模式
│   ├── 财务分析
│   └── 核心团队
├── 投资价值与风险（Agent D）
└── 附录（Agent E）
    ├── 财务模型
    └── 可比公司分析
```

#### 3.2 多Agent协作流程

```mermaid
思路示意：
输入资料 → 主控Agent（任务拆解）
              ↓
    ┌─────────┼─────────┐
    ↓         ↓         ↓
 Agent 1   Agent 2   Agent 3
(行业)    (公司)    (财务)
    ↓         ↓         ↓
    └─────────┴─────────┘
              ↓
       主控Agent（整合润色）
              ↓
       Obsidian人工审核
```

#### 3.3 数据驱动的分析能力

- 财务数据自动提取与可视化
- 可比公司数据对比分析
- 敏感性分析与情景模拟

#### 3.4 Obsidian：最终编辑与知识沉淀

**为什么用Obsidian？**
- Markdown原生支持，与AI输出无缝衔接
- 双向链接，构建个人投研知识图谱
- 本地存储，数据安全可控
- 插件丰富，可扩展性强

**工作流**：
```
AI初稿 → Obsidian编辑 → 内部审核 → 定稿输出
             ↓
      标签归档、关联历史项目
```

### 模块4：最终输出与交付（45分钟）

#### 4.1 精美PDF生成

**工具链**：
- Markdown → Typora / MarkText（所见即所得编辑）
- 专业排版：Pandoc + LaTeX模板
- 自动化：Python脚本批量生成

**设计要点**：
- 投资级报告的专业排版
- 图表清晰、配色统一
- 页眉页脚、目录自动生成

#### 4.2 AI生成PPT

**推荐工具**：
- Gamma：内容优先的演示设计
- Tome：AI叙事结构优化
- 讯飞智文：中文场景适配

**提示词技巧**：
- 明确页数、风格、核心论点
- 提供品牌VI要求（配色、字体）
- 迭代优化而非一次性完美

## 六、常见问题FAQ

**Q1：私有化部署成本如何？**
A：RAGFlow开源免费，最低配置需8GB内存服务器。个人可先本地Docker部署试用，团队再根据规模考虑云端部署。

**Q2：AI写的研报能用吗？会不会有幻觉？**
A：AI适合"初稿+框架"，关键数据必须人工复核。建议采用RAG确保有源可溯，重要结论交叉验证。

**Q3：学习曲线陡峭吗？**
A：工具操作1-2天可上手，但"与AI协作"的思维转变需要1-2个月实践。建议从简单任务开始，逐步建立信任。

**Q4：团队如何协同使用？**
A：建议统一知识库分类标准、Prompt模板规范。RAGFlow支持多用户权限管理，可实现团队协作。
