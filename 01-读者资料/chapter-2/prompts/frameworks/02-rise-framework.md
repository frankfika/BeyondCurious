## 框架定义

RISE 框架聚焦任务执行的核心要素。

| 要素 | 英文 | 中文 | 说明 |
|------|------|------|------|
| R | Role | 角色 | 确定 AI 的专业身份 |
| I | Input | 输入 | 提供必要的原始信息 |
| S | Step | 步骤 | 详细描述任务执行过程 |
| E | Expectation | 期望 | 明确最终结果标准 |

## 应用示例：会议记录助手

```markdown
# Role
董事长助理，专注于高效、准确地整理会议纪要。

# Input
用户提供的会议基本信息，包括会议主题、时间、参会人员、议程、讨论内容、决定及行动计划等。

# Step
1. 引导用户提供完整的会议基本信息
2. 按结构整理信息（主题、时间、参会人员、议程、讨论内容、决定与行动计划、下一步计划）
3. 对信息进行语句微调，保证简洁准确
4. 输出结构清晰、重点突出的会议纪要

# Expectation
生成的会议纪要内容准确无误，重点明确，格式规范，语言简洁专业。
```

## 使用方法

1. 定义角色：给 AI 分配专业角色
2. 提供输入：说明所需输入信息
3. 描述步骤：列出执行步骤
4. 明确期望：说明成功标准

## 优势

- 角色驱动，激活专业能力
- 步骤清晰，确保任务完成
- 结果导向，明确成功标准

## 适用场景

- 会议记录整理
- 文档摘要生成
- 数据分析报告
- 项目管理任务
